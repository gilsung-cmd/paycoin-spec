페이코인, 1분기 결제 거래량 전년 동기 대비 230% 성장
연합뉴스 2026.04.30
원문 보기 ↗

페이코인(대표 한정수)이 2026년 1분기 결제 거래량이 전년 동기 대비 230% 성장했다고 30일 밝혔다. 이는 온라인 결제 가맹점 확대와 오프라인 결제 활성화에 따른 결과다.

페이코인 관계자는 "올해 들어 주요 편의점 체인과 외식 프랜차이즈를 중심으로 가맹점이 빠르게 확대되고 있으며, PCI 토큰 결제의 편의성을 경험한 사용자들의 재이용률이 높게 나타나고 있다"고 말했다.

회사 측은 2026년 하반기까지 가맹점 3만 개소 확대를 목표로 마케팅 투자를 지속할 계획이라고 밝혔다. PCI 토큰 시세도 이번 발표에 힘입어 소폭 상승세를 보였다.

SCREEN_HOME P1
메인 홈
앱 실행 후 첫 진입 화면. 자산 현황 요약, 롤링 배너, 타일 메뉴, 최근 소식으로 구성. 모든 주요 기능으로의 진입 허브 역할.
⚠️ 개발 주의사항
타일 메뉴 하드코딩 절대 금지. 서버 JSON을 파싱해 렌더링. 타일 수·순서·노출 여부 모두 어드민에서 관리.
롤링 배너는 자동 3초 순환 + 수동 스와이프 모두 지원. 배너 없으면 영역 자체 미표시(레이아웃 재배치).
자산 합계 API 실패 시 "자산 정보를 불러오지 못했습니다 + 다시 시도" — 빈 화면 노출 금지.
앱 실행 시 로컬 캐시를 즉시 표시 → 백그라운드 API 완료 시 실시간 값으로 교체(애니메이션 처리).
롤링 배너
자동 3초 롤링, 인디케이터 점 표시. 배너 이미지 로드 실패 시 기본 플레이스홀더 표시.
탭 → 지정 딥링크 또는 외부 웹뷰 이동. 링크 없으면 탭 이벤트 무반응.
GET /api/v1/home/banners → Banner[]
타일 메뉴 구성 규칙
1행: 1/3 너비 × 3개 / 2행: 2/3(넓은 타일) + 1/3 / 3행: 가로 스크롤, 우측 끝 타일이 화면에 걸쳐 '더 있음'을 암시.
비활성 타일 제거 후 나머지 레이아웃 자동 재배치. 네트워크 오류 시 마지막 캐시 구성 사용.
GET /api/v1/home/tiles → Tile[] tile.row_type: "1x1" | "2x1" | "scroll"
자산 합계 영역
합계 = 자체 지갑 + 모든 연결 지갑 KRW 합산 (실시간 시세 기준)
⚠️ 변동 금액·변동률 기준 시점 미확정 (전일 0시 vs 마지막 로그인) — 필드 예약, null 시 UI 미표시
is_stale=true: 금액 옆 ⚠ 아이콘 + "시세가 최신이 아닐 수 있습니다" 안내
영역 탭 → 지갑 화면 / "자산현황 보기" 탭 → 자산현황 화면
알림 뱃지
미확인 알림 수 표시, 99 초과 시 "99+" 고정. 포그라운드 전환 시 1회 재조회.
GET /api/v1/notifications/unread-count
최근 소식
최신 소식 5건(게시일자 DESC). "전체 보기" → 소식 리스트 화면
GET /api/v1/news?limit=5
SCREEN_WALLET P1
지갑
보유 코인 잔액·시세를 코인 단위로 실시간 조회. "지금 내가 얼마를 갖고 있나"에 집중. 분석·비중은 자산현황 화면에서 제공(역할 분리).
⚠️ 개발 주의사항
자체 지갑 PCI는 항상 최상단 고정. 연결 지갑은 connected_at ASC 정렬.
연결 지갑이 없는 거래소 탭: 회색 + "연결하기" 버튼 → WalletConnect 플로우 진입.
▲(상승) 빨간색, ▼(하락) 파란색 — 한국 주식 UI 관행. FLAT은 회색 '-'.
아이콘 표시 규칙
자체 지갑: 코인 심볼 아이콘만 (거래소 오버레이 없음)
연결 지갑: 코인 아이콘 + 거래소 로고 우하단 오버레이 (24px)
동일 코인 복수 거래소: 각각 별도 행으로 독립 표시
탭 필터 로직
전체 | 내 지갑(SELF) | 빗썸 | 코인원 | 코빗 GET /api/v1/wallets?source={tab_filter}
SCREEN_WALLET_DETAIL (자체 PCI) P1
지갑 상세 — PCI 자체 지갑
PCI 자체 지갑 전용 화면. 출금하기 버튼, PCI 정보 보기, 최근 거래 내역(5건)이 이 화면에서만 표시됨.
⚠️ 개발 주의사항
출금하기 버튼: 자체 지갑(wallet_type=SELF, coin=PCI)에만 표시. 다른 지갑 유형에서는 절대 노출 안 됨.
최근 거래 내역도 자체 지갑에만 표시. 연결 지갑은 "거래소 앱에서 확인" 안내문으로 대체.
referrer 파라미터로 뒤로가기 레이블 동적 변경 (지갑/자산현황/PCI거래소상세 등).
출금 플로우 검증
PCI 주소 형식 regex 실시간 검증. 자기 자신 주소 출금 불가.
amount + fee ≤ balance 검증. 최소 출금 수량 이상.
PIN/생체인증 → 최종 확인 팝업 → 네트워크 제출.
햅틱: PIN 입력 Light / PIN 오류 Error(강한 진동).
API 명세
GET /api/v1/wallets/{id} GET /api/v1/wallets/{id}/transactions?limit=5 GET /api/v1/wallets/{id}/withdraw-info POST /api/v1/wallets/{id}/withdraw
SCREEN_WALLET_DETAIL (연결 지갑) P1
지갑 상세 — 연결 지갑 (PCI/기타 코인)
WalletConnect로 연결된 거래소 지갑 상세. 출금하기 없음. "거래소로 이동하기" 버튼 제공. 거래 내역은 거래소 앱에서 확인 안내.
⚠️ 개발 주의사항
출금하기 버튼 없음. 대신 "거래소로 이동하기" 버튼 표시.
거래 내역 미표시. "거래 내역은 거래소 앱에서 확인" 안내 영역으로 대체.
거래소 이동: 거래소 앱 딥링크 우선, 미설치 시 웹 URL fallback.
코인 정보 버튼
PCI 거래소 연결 지갑: "PCI 정보" 버튼 없음 (자체 지갑만 제공)
기타 코인(BTC/ETH 등): "[심볼명] 정보" 버튼 → SCREEN_TOKEN_DETAIL(exchange, coin_symbol)
2차 주소 표시 조건
XRP: Destination Tag / 빗썸 PCI·코인원 PCI: Memo / 기타: 미표시
SCREEN_ASSET_STATUS P2
자산현황
포트폴리오 변동 추이·비중·손익 분석. 지갑 화면이 실시간 잔액이라면, 이 화면은 시각화된 분석 제공. 두 화면의 역할을 명확히 분리할 것.
⚠️ 미확정 항목
⚠️ 변동률 기준 시점 미확정 (전일 0시 vs 마지막 로그인) — change_amount, change_rate 필드 예약
⚠️ 자산 표시 UX: 캐시→실시간 Toss 방식 구현 방법 개발팀 협의 필요
⚠️ 그래프 제공 기간 구간 미확정 (1D/1W/1M/3M) — period 파라미터 예약
비중 그래프 섹션 조건
코인 2종+ → 자산별 탭 + 거래소별 탭 모두 표시
코인 1종 + 거래소 2개+ → 거래소별 탭만 표시
코인 1종 + 거래소 1개 → 섹션 전체 미표시. 서버가 show_by_asset/show_by_exchange boolean으로 판단해서 내려줄 것.
API 명세
GET /api/v1/asset-status/chart?period=1D GET /api/v1/asset-status/distribution → show_by_asset, show_by_exchange, by_asset[], by_exchange[] GET /api/v1/asset-status/wallets

③ 전날 참여 + 결과 집계 완료 → 투표 영역 상단에 전날 결과 카드

결제
QR코드 결제와 오프라인 바코드 결제를 하나의 진입점으로 통합. 이번달 누적 결제 금액·혜택 금액, 방식 선택 버튼, 가맹점 혜택 슬라이드, 최근 결제 내역 5건 표시.
⚠️ 개발 주의사항
QR코드 결제 탭 시: 카메라 권한 확인 → 없으면 권한 요청 → 거부 시 설정 이동 안내. 카메라 뷰파인더 → QR 인식 → 기존 플로우. 취소/뒤로가기 → 결제 메인 복귀.
이벤트 배너 없으면 섹션 전체 미표시.
가맹점 슬라이드: 자동 슬라이딩(3초 간격) 무한 반복 + 수동 스와이프.
⚠️ 할인 금액 집계 기준 및 데이터 제공 방식 협의 필요 — monthly_discount_krw 필드 예약
표시 문구 정책
결제 현황 레이블: "이번달 누적 결제 금액"
할인 문구: "이번달 {N}원 혜택을 받았어요!" (0원 시 미표시)
최근 결제 내역
최근 5건 표시. "전체 보기" 탭 → 거래내역 화면으로 이동.
할인 금액: 있는 경우만 표시, 없으면 해당 영역 미표시.
GET /api/v1/payment/monthly-summary GET /api/v1/payment/merchants GET /api/v1/payment/recent?limit=5
